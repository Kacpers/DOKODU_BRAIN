Rekomendacja dla kursu n8n
Use case: płatny kurs online z dużą ilością kodu, diagramami ASCII, notatkami prowadzącego, output: standalone HTML dla studentów

Kiedy wybrać Slidev

  • Animacje v-click — każdy krok workflow pojawia się osobno
  • Shiki Magic Move — animowana ewolucja kodu (idealna dla n8n Code Node)
  • Wbudowane Mermaid — renderuje diagramy przepływu
  • Monaco Editor — live coding bezpośrednio w slajdzie
  • Profesjonalna jakość wizualna bez dodatkowego CSS
  • Najnowocześniejszy tooling — aktywny development

Kiedy wybrać reveal-md

  • Potrzebujesz JEDNEGO folderu --static gotowego offline
  • Wolisz prostszy setup (zero Vue/Vite)
  • Chcesz line-by-line code highlighting bez JS komponentów
  • Bardziej znasz reveal.js ekosystem
  • Dystrybucja jako prosty folder ZIP bez serwera Node
Verdict dla kursu n8n: Pierwsze wybór to Slidev — Shiki Magic Move animuje zmiany w kodzie JS (idealne do pokazywania refaktoryzacji Code Node), v-click buduje workflow krok po kroku, Mermaid renderuje diagramy przepływu n8n. Jakość wizualna jest na poziomie płatnego kursu out-of-the-box.

Jedyne zastrzeżenie: Slidev nie eksportuje do jednego pliku .html — eksportuje folder dist/. Możesz go spakować do ZIP i kazać studentom odpalić przez python -m http.server lub hostować na GitHub Pages / własnym serwerze. Jeśli potrzebujesz bezwzględnie jednego pliku HTML — użyj reveal-md z opcją --static (generuje self-contained folder) lub Marp (generuje jeden .html, ale bez zaawansowanych animacji).

Kluczowe zastrzeżenia

!
Slidev: brak single-file HTML. slidev build generuje folder SPA (wiele plików). Nie da się otworzyć bezpośrednio jako plik .html w przeglądarce bez serwera HTTP lub bez wypakowywania archiwum. Jeśli studenci mają otwierać podwójnym kliknięciem — wybierz Marp lub reveal-md.
!
Slidev: animacje a eksport PDF. Domyślnie PDF nie zawiera kroków animacji. Użyj flagi --with-clicks żeby każdy v-click stał się osobną stroną PDF. Dla studentów lepiej dystrybuować HTML niż PDF.
i
ASCII diagramy we wszystkich narzędziach: ASCII art działa wszędzie jako blok kodu z fontami monospace. Żadne z narzędzi nie renderuje ASCII jako SVG — po prostu wyświetla jako tekst w pre. Dla przepływów n8n rozważ Mermaid (Slidev/Marp) który renderuje ładne diagramy z bloków tekstu.
i
WSL / Linux: Wszystkie narzędzia oparte na Node.js działają identycznie na Linux i WSL2. PDF export w Slidev i Marp wymaga Chromium (Playwright/Puppeteer) — na WSL może wymagać dodatkowych zależności systemowych (apt-get install chromium-browser).
